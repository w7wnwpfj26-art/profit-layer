# 安全與負責披露

本專案涉及自動化 DeFi 策略與錢包操作，**安全風險等級高**。請在部署與使用前仔細閱讀本文件。

## 1. 安全報告（負責披露）
- 請使用 GitHub Security Advisories 提交安全報告（推薦）。
- 請勿在公開 Issue 中披露安全漏洞細節。
- 我們會在合理時間內回覆並協調修復與披露時間。

## 2. 支援範圍
- 僅保障 **主分支（`main`）** 與 **最新版本**。
- 歷史版本僅在必要時提供修復建議，不保證補丁。

## 3. 密鑰管理
- **嚴禁**提交 `.env` 或任何密鑰文件到版本庫。
- 使用 `WALLET_ENCRYPTION_KEY` 加密存儲私鑰（AES-256-GCM）。
- `JWT_SECRET` 必須為強隨機字串（建議 ≥ 32 字符）。

## 4. API 安全
- 所有寫入 API 必須啟用認證（`lib/auth.ts`）。
- 建議開啟 Rate Limit（`lib/rateLimit.ts`）。
- SQL 查詢必須使用參數化語句，避免注入風險。

## 5. 交易與風控
- 緊急停止開關（Kill Switch）必須可用。
- 單筆限額 + 每日限額雙重控制。
- 上鏈前務必使用模擬交易（Tx Simulator）。
- 支援 Dry Run 模式避免真實交易。

## 6. 前端安全
- 敏感輸入使用遮罩與局部顯示。
- 配置校驗應由前後端同時保障（前端：`app/lib/utils.ts`）。

## 7. 依賴與合規
- 定期掃描依賴漏洞（建議：`pnpm audit`）。
- 確保第三方 API 的服務條款允許自動化交易用途。
