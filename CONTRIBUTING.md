# 贡献指南

感谢你对本项目的兴趣！为了让协作高效、稳定且安全，请遵循以下规范。

## 1. 开发前准备
- 确保本地使用 Node.js >= 20 与 pnpm >= 9
- 使用 `cp .env.example .env` 创建本地配置
- **严禁**提交 `.env` 或密钥相关文件

## 2. 分支与提交
- 建议使用 `feature/xxx` 或 `fix/xxx` 作为分支命名
- 提交信息保持清晰、可读

## 3. 代码风格
- 遵循项目现有的 ESLint/TypeScript 风格
- UI 改动需保持一致的视觉系统与组件风格

## 4. 测试与验证
- 若修改核心逻辑，请补充或更新测试
- 确保项目可正常启动与构建

## 5. Pull Request 要求
- 描述改动动机、影响范围与测试方式
- 明确是否涉及安全敏感内容
- PR 合并前需通过 Review

## 6. 安全说明
- 如发现安全问题，请不要提交公开 Issue
- 参考 `docs/SECURITY.md` 进行负责披露
